Native Types

Primitives

  1. Integer numbers
    1. s8 (1 byte, signed)
    2. u8 (1 byte, unsigned)
    3. s16 (2 bytes, signed)
    4. u16 (2 bytes, unsigned)
    5. s32 (4 bytes, signed)
    6. u32 (4 bytes, unsigned)
    7. s64 (8 bytes, signed)
    8. u64 (8 bytes, unsigned)
    9. char (equivalent to u8)
    10. short (equivalent to s16)
    11. int (equivalent to s32)
    12. long (equivalent to s64)
    13. c_short (for ABI compatibility with C)
    14. c_ushort (for ABI compatibility with C)
    15. c_int (for ABI compatibility with C)
    16. c_uint (for ABI compatibility with C)
    17. c_long (for ABI compatibility with C)
    18. c_ulong (for ABI compatibility with C)
    19. c_longlong (for ABI compatibility with C)
    20. c_ulonglong (for ABI compatibility with C)
    21. c_longdouble (for ABI compatibility with C)
  2. Booleans
    1. bool (1 byte)
  3. Floating-Point Numbers
    1. f32 (4 bytes)
    2. f64 (8 bytes)
    3. float (equivalent to f32)
    4. double (equivalent to f64)
  4. Unvalued
    1. null
    2. noreturn
    3. c_void (for ABI compatibility with C)
  5. Miscellaneous
    1. type

Number Literals

Unless a type is specified, numbers are assumed to be the smallest type that can store the literal.

const a = 128; // U8
const b = 128.0; // float
const c = 123.; // float
const d: float = 128; // float
const e = 128 as float; // float

String Literals

Catlang uses the C-style convention that character literals are wrapped with single quotes, while string literals are wrapped with double quotes:

const a = 'a'; // char
const b = "a"; // string
const c = "abc"; // string
const d = 'abc'; // ERROR!

A notable feature of the language is string interpolation:

let name = "";
print("Enter your name: ");
readLine(name);
print("Hello, ${name}!\n");

Pointers

To retrieve the pointer to an object, use the @ operator. To dereference a pointer, use the * operator.

Functions can be called on pointers the same way they can on standard objects - there's no need for C++'s arrow operator.
